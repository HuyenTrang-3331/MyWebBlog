Hiểu rõ về Closure trong JavaScript – Bí mật đằng sau sức mạnh của hàm

🕓 Đăng ngày 10/09/2025

🧠 “Closure – nền tảng cho sức mạnh và sự linh hoạt của JavaScript.”

Closure là một trong những khái niệm cốt lõi và quan trọng nhất của JavaScript.
Nó là thứ làm cho JS có thể thực hiện các tính năng như callback, module, private variable, và xử lý bất đồng bộ mượt mà.

📘 1. Closure là gì?

Closure xuất hiện khi một hàm được định nghĩa bên trong một hàm khác
và hàm con có thể truy cập các biến của hàm cha,
ngay cả sau khi hàm cha đã kết thúc thực thi.

function outer() {
  let message = "Xin chào từ outer!";
  function inner() {
    console.log(message);
  }
  return inner;
}

const greet = outer();
greet(); // 👉 "Xin chào từ outer!"

📌 Biến message không bị mất khi outer() chạy xong,
vì inner() vẫn “đóng gói” (closure) biến đó lại trong bộ nhớ.

🔍 2. Cơ chế hoạt động của Closure

Khi JavaScript thực thi một hàm, nó tạo ra một Execution Context mới.
Mỗi hàm đều có phạm vi của riêng nó, nhưng hàm con kế thừa phạm vi của hàm cha thông qua Scope Chain.

Global Scope
 └── outer() Scope
      └── inner() Scope

💡 Mỗi khi outer() chạy, message được tạo mới.
Hàm inner() tham chiếu đến message thông qua Lexical Scope (phạm vi tĩnh theo vị trí viết trong code).

⚙️ 3. Vì sao Closure quan trọng?

Closure giúp duy trì trạng thái, bảo mật dữ liệu, và xây dựng module riêng biệt.
Dưới đây là một số ứng dụng điển hình:

Ứng dụng Mô tả Ví dụ
Giữ trạng thái (State) Lưu dữ liệu giữa các lần gọi hàm counter()
Ẩn thông tin (Encapsulation) Giữ biến private, tránh truy cập ngoài ý muốn createUser()
Callback / Async logic Truy cập biến cha trong async function setTimeout()
Function Factory Tạo ra các hàm cấu hình động makeMultiplier(5)

🔢 4. Ví dụ: Tạo bộ đếm bằng Closure

function counter() {
  let count = 0; // biến "private"
  return function() {
    count++;
    console.log("Giá trị hiện tại:", count);
  };
}

const add = counter();
add(); // 1
add(); // 2
add(); // 3

🎯 Mỗi lần gọi add(), hàm vẫn “nhớ” được giá trị count dù counter() đã xong.

🧩 5. Closure trong vòng lặp

Một lỗi kinh điển khi dùng var trong vòng lặp:

for (var i = 1; i <= 3; i++) {
  setTimeout(() => console.log(i), 1000);
}
// 👉 In ra: 4, 4, 4

💥 Vì var không có block scope, chỉ có một biến i duy nhất.

✅ Cách sửa với let (tạo phạm vi riêng cho từng vòng lặp):

for (let i = 1; i <= 3; i++) {
  setTimeout(() => console.log(i), 1000);
}
// 👉 In ra: 1, 2, 3

⚡ 6. Closure trong thực tế (ứng dụng nâng cao)

🔹 6.1 – Duy trì trạng thái trong module

function createUser(name) {
  let score = 0;
  return {
    getName: () => name,
    increase: () => score++,
    getScore: () => score
  };
}

const user = createUser("Trang");
user.increase();
user.increase();
console.log(`${user.getName()} có điểm: ${user.getScore()}`);

📜 Kết quả:

Trang có điểm: 2

🔹 6.2 – Function Factory

function makeMultiplier(x) {
  return function(y) {
    return x * y;
  };
}

const double = makeMultiplier(2);
const triple = makeMultiplier(3);

console.log(double(5)); // 10
console.log(triple(5)); // 15

💡 Mỗi hàm double và triple giữ lại biến x khác nhau.

🚀 7. Closure và bất đồng bộ (Async)

Closure giúp async function truy cập đúng giá trị tại thời điểm gọi.

function fetchData(url) {
  setTimeout(() => {
    console.log(`Lấy dữ liệu từ ${url}`);
  }, 1000);
}

fetchData("https://api.github.com/users/trang");

Ở đây, url vẫn tồn tại trong closure của callback setTimeout,
nên dù 1 giây sau, giá trị url vẫn chính xác.

🧱 8. Các lỗi thường gặp với Closure

Lỗi Nguyên nhân Cách khắc phục
Lưu sai giá trị trong vòng lặp Dùng var khiến biến bị ghi đè Dùng let hoặc IIFE
Giữ quá nhiều biến trong closure Không giải phóng biến → rò rỉ bộ nhớ Đặt closure trong phạm vi giới hạn
Không hiểu phạm vi từ vựng (Lexical Scope) Closure truy cập theo vị trí khai báo, không theo nơi gọi Hiểu rõ scope chain

🧮 9. Hình minh họa cơ chế Closure

Global Scope
 ├── function outer()
 │     ├── biến: message
 │     └── function inner()
 │           └── có thể truy cập message
 └── inner() được trả ra ngoài → vẫn giữ link đến message

💡 Closure = Hàm + Môi trường nơi nó được tạo ra.

🧾 10. Tổng kết kiến thức

Thành phần Vai trò Ví dụ
Scope Chain Xác định phạm vi truy cập biến let, const
Closure Hàm nhớ biến của hàm cha counter()
Private Variable Biến chỉ tồn tại trong closure score trong createUser()
Function Factory Tạo ra hàm mới dựa vào tham số makeMultiplier(3)
Async Callback Ghi nhớ biến khi chạy bất đồng bộ setTimeout()

🎯 Kết luận

  • Closure là một trong những khái niệm quan trọng nhất của JavaScript.
  • Nó giúp hàm nhớ được môi trường ban đầu,
    từ đó làm nên các kỹ thuật như: Module Pattern, Callback, Async/Await, Factory Function.
  • Nếu hiểu rõ Closure, bạn sẽ tư duy đúng về scope, memory, và lập trình bất đồng bộ.
