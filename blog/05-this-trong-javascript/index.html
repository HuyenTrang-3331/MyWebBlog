Từ khóa this trong JavaScript – Khi nào trỏ về đâu?

🕓 Đăng ngày 25/09/2025

💡 “Nếu không hiểu this, bạn đang code JavaScript mà không hiểu JavaScript.”

🧭 1. this là gì?

this đại diện cho ngữ cảnh thực thi (execution context) hiện tại – nó không cố định mà phụ thuộc vào cách hàm được gọi, chứ không phải nơi hàm được định nghĩa.

🧩 2. this trong phạm vi toàn cục

console.log(this === window); // true (trình duyệt)

Trong Node.js:

console.log(this === {}); // true trong module

⚙️ 3. this trong hàm thường

Trong strict mode, this mặc định là undefined nếu hàm không thuộc object nào.

function show() {
  console.log(this);
}
show(); // undefined (vì strict mode)

Nếu không có strict mode, this sẽ mặc định là window.

🧱 4. this trong object

Khi hàm được gọi qua object, this trỏ đến object đó.

const user = {
  name: "Trang",
  sayHi() {
    console.log("Hi " + this.name);
  }
};
user.sayHi(); // Hi Trang

🔹 Ở đây, this chính là user.

⚡ 5. this trong Arrow Function

Arrow function không có this riêng.
Nó kế thừa this từ ngữ cảnh bên ngoài (lexical scope).

const obj = {
  name: "Trang",
  show: () => console.log(this.name)
};
obj.show(); // undefined

this ở đây không phải obj, mà là window (hoặc undefined trong strict mode).

🧠 6. this trong Class

this trong class luôn trỏ đến đối tượng được tạo từ class đó (instance).

class Person {
  constructor(name) {
    this.name = name;
  }

  sayHi() {
    console.log("Hi " + this.name);
  }
}

const trang = new Person("Trang");
trang.sayHi(); // Hi Trang

🔄 7. this khi mất ngữ cảnh

Nếu tách hàm ra khỏi object, this sẽ mất ngữ cảnh ban đầu.

const user = { name: "Trang", hi() { console.log(this.name); } };
const hiFn = user.hi;
hiFn(); // undefined

✅ Cách khắc phục:

  • Dùng bind(), call(), apply()
  • Hoặc dùng Arrow Function để giữ ngữ cảnh
const hiBound = user.hi.bind(user);
hiBound(); // Trang

🧩 8. this trong sự kiện DOM

<button id="btn">Click me</button>

<script>
document.getElementById("btn").addEventListener("click", function() {
  console.log(this); // <button> element
});
</script>

Khi dùng function thông thường, this trỏ đến element.
Nếu dùng arrow function, this không còn là element nữa.

⚙️ 9. this trong callback và bất đồng bộ

const obj = {
  name: "Trang",
  start() {
    setTimeout(function() {
      console.log(this.name);
    }, 1000);
  }
};
obj.start(); // undefined

✅ Khắc phục bằng Arrow Function:

const obj = {
  name: "Trang",
  start() {
    setTimeout(() => {
      console.log(this.name);
    }, 1000);
  }
};
obj.start(); // Trang

📚 10. Tổng kết

Ngữ cảnh this trỏ đến Ví dụ
Toàn cục window / undefined console.log(this)
Hàm thường undefined hoặc window show()
Object Object chứa hàm user.sayHi()
Arrow Function Ngữ cảnh bên ngoài () => this.name
Class Instance new Person()
DOM Event Element được click button.onclick = function() {}

🎯 Kết luận

  • this không phụ thuộc vào nơi định nghĩa, mà vào cách gọi hàm.
  • Arrow Function kế thừa this từ cha.
  • bind(), call(), apply() giúp bạn kiểm soát this chính xác.
  • Hiểu rõ this = code JavaScript tự tin và ít lỗi hơn 🚀
