Hiểu sâu về Event Loop trong JavaScript – Cơ chế bất đồng bộ mượt mà

🕓 Đăng ngày 05/09/2025

⚡ “Hiểu Event Loop – nắm linh hồn JavaScript.”

JavaScript là ngôn ngữ đơn luồng (single-thread), nghĩa là chỉ có một luồng thực thi duy nhất.
Thế nhưng, mỗi khi ta gọi setTimeout(), fetch(), hay thao tác DOM, mọi thứ vẫn diễn ra “song song” mượt mà.
Bí mật nằm ở cơ chế Event Loop.

🧠 1. JavaScript là ngôn ngữ đơn luồng

JavaScript chỉ có một Call Stack – nơi mọi đoạn mã được thực thi tuần tự.
Nếu một đoạn code chiếm dụng quá lâu, nó sẽ “chặn” toàn bộ chương trình.

console.log("Bắt đầu");
for (let i = 0; i < 1e9; i++) {} // vòng lặp cực lớn
console.log("Kết thúc");

💥 Khi chạy, trình duyệt sẽ bị đứng vài giây vì Call Stack bị chiếm toàn bộ.

⚙️ 2. Thành phần trong cơ chế Event Loop

Cơ chế hoạt động dựa trên ba thành phần chính:

Thành phần Vai trò
Call Stack Thực thi các lệnh đồng bộ.
Web APIs Xử lý các tác vụ bất đồng bộ (như setTimeout, fetch, DOM events, …).
Task Queues Nơi chứa callback sẵn sàng được thực thi – gồm Microtask & Macrotask.

🔁 3. Mô phỏng hoạt động tổng quát

┌──────────────────────────────┐
│          Call Stack          │ ← Chạy từng lệnh JS
└──────────────┬───────────────┘
               │
               ▼
┌──────────────┐
│   Web APIs   │ ← Thực thi async (setTimeout, fetch)
└──────┬───────┘
       ▼
┌──────────────────────────────┐
│       Task Queues            │
│ - Microtask Queue (Promise)  │
│ - Macrotask Queue (Timeout)  │
└──────────┬───────────────────┘
           ▼
     ┌────────────┐
     │ Event Loop │ ← Kiểm tra Stack & đẩy callback từ Queue lên Stack
     └────────────┘

🔄 Event Loop liên tục “quan sát” Stack. Khi Stack rỗng, nó lấy callback trong Queue để thực thi.

🧩 4. Ví dụ cơ bản: setTimeout và Promise

console.log("A");
setTimeout(() => console.log("B"), 0);
Promise.resolve().then(() => console.log("C"));
console.log("D");

📜 Kết quả:

A
D
C
B

Giải thích:

  1. A và D chạy đồng bộ trên Call Stack.
  2. setTimeout() đẩy callback B sang Web API → chờ → vào Macrotask Queue.
  3. Promise.then() đẩy callback C vào Microtask Queue.
  4. Khi Stack trống → Event Loop lấy Microtask C chạy trước → sau đó mới đến Macrotask B.

⚖️ 5. Microtask vs Macrotask

Loại tác vụ Ví dụ Khi thực thi
Microtask Promise.then(), queueMicrotask() - Ngay sau khi Stack trống
Macrotask setTimeout, setInterval, I/O Sau khi tất cả Microtask hoàn thành 
setTimeout(() => console.log("timeout"), 0);
Promise.resolve().then(() => console.log("promise"));
console.log("main");

📜 Kết quả:

main
promise
timeout

⚡ Microtask (Promise) luôn có độ ưu tiên cao hơn Macrotask (Timeout).

🧱 6. Vấn đề Blocking và cách khắc phục

Đoạn code sau làm “đơ” giao diện người dùng:

for (let i = 0; i < 1e9; i++) {}
console.log("Hoàn thành");

✅ Cách khắc phục — chia nhỏ công việc:

let count = 0;
function process() {
  for (let i = 0; i < 1e7; i++) {
    count++;
  }
  console.log("Đã xử lý:", count);
  if (count < 1e9) setTimeout(process, 0);
}
process();

Nhờ chia nhỏ bằng setTimeout, mỗi phần được xử lý ở vòng Event Loop khác nhau → giao diện vẫn phản hồi mượt mà.

⚡ 7. Ví dụ nâng cao: Promise lồng nhau và Timeout

console.log("1: Start");

setTimeout(() => console.log("2: Timeout 1"), 0);

Promise.resolve()
  .then(() => {
    console.log("3: Promise 1");
    return Promise.resolve();
  })
  .then(() => console.log("4: Promise 2"));

setTimeout(() => console.log("5: Timeout 2"), 0);

console.log("6: End");

📜 Kết quả:

1: Start
6: End
3: Promise 1
4: Promise 2
2: Timeout 1
5: Timeout 2

🧠 Giải thích:

  • Promise nằm trong Microtask Queue, chạy trước tất cả Timeout (Macrotask).
  • Event Loop ưu tiên chạy toàn bộ Microtask trước khi chuyển sang Macrotask.

🧮 8. Event Loop trong Node.js

Node.js cũng có Event Loop, nhưng chia thành 6 giai đoạn chính:

  1. Timers → Xử lý setTimeout, setInterval.
  2. Pending callbacks → Các callback bị hoãn.
  3. Idle, prepare → Nội bộ Node.js.
  4. Poll → Đợi các I/O event mới.
  5. Check → Xử lý setImmediate().
  6. Close callbacks → Dọn dẹp kết nối.

🔹 Giữa mỗi giai đoạn, Node.js cũng xử lý Microtask Queue giống Promise trong trình duyệt.

🌐 9. Mô phỏng thực tế: gọi API và cập nhật UI

console.log("1. Gọi API...");

fetch("https://api.github.com/users/octocat")
  .then(() => console.log("2. Dữ liệu đã về"))
  .then(() => console.log("3. Cập nhật UI"));

console.log("4. Người dùng vẫn thao tác được");

📜 Kết quả:

1. Gọi API...
4. Người dùng vẫn thao tác được
2. Dữ liệu đã về
3. Cập nhật UI

🎯 Nhờ Event Loop, người dùng vẫn có thể click, nhập liệu, và cuộn trang trong khi JavaScript đang chờ phản hồi mạng.

🧾 10. Tổng kết kiến thức

Thành phần Vai trò Ví dụ
Call Stack Thực thi lệnh đồng bộ console.log()
Web APIs Xử lý bất đồng bộ fetch(), setTimeout()
Microtask Queue Lưu callback Promise .then(), await
Macrotask Queue Lưu các callback lớn setTimeout, I/O, UI event
Event Loop Giám sát & điều phối Kiểm tra Stack, đẩy tác vụ vào thực thi

🎯 Kết luận

  • JavaScript chỉ có một luồng, nhưng nhờ Event Loop mà có thể chạy hàng loạt tác vụ bất đồng bộ mượt mà.
  • Microtask (Promise, async/await) luôn chạy trước Macrotask (setTimeout, event).
  • Nắm vững Event Loop giúp:
    • Viết code không bị chặn.
    • Tối ưu hiệu năng web.
    • Hiểu sâu về hoạt động của Node.js & trình duyệt.
